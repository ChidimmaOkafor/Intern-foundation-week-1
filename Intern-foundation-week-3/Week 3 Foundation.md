# Week 3 Engineering Foundation

## Overview

Week 3 focused on improving code quality, debugging skills, and technical documentation. The objective was to learn how to write clean, maintainable code, systematically identify and fix bugs, document engineering work, and communicate technical findings clearly.

## What I Learned

During this week, I learned how to:

* Apply clean code principles to improve readability.
* Refactor existing code without changing its behavior.
* Debug programs using a structured approach instead of guessing.
* Identify different types of programming errors.
* Write clear bug reports and technical documentation.
* Test code after making changes to ensure everything works correctly.

## Topics Covered

* Clean Code Principles
* Code Refactoring
* Debugging Techniques
* Problem Solving
* Bug Reporting
* Technical Documentation
* Markdown Documentation

## Code Refactoring Summary

I refactored Python programs by improving variable names, simplifying the code structure, removing unnecessary repetition, and making the code easier to understand while keeping the original functionality unchanged.

## Debugging Practice Summary

I completed debugging exercises by identifying syntax, runtime, and logical errors. For each issue, I analyzed the problem, applied the appropriate fix, tested the solution, and documented what caused the bug and how it was resolved.

## Bug Report Summary

I created a bug report describing the issue, the steps to reproduce it, the expected and actual results, the root cause, the implemented fix, and the outcome after testing.

## How I Tested My Work

* Ran each Python program after making changes.
* Verified that all expected outputs were produced.
* Re-tested programs after fixing bugs.
* Confirmed that refactoring did not change program behavior.
* Reviewed documentation for clarity and completeness.
