# Debugging Notes

## What is Debugging?

Debugging is the process of finding, understanding, and fixing problems (bugs) in software.

Debugging is not guessing. A developer should follow a systematic process to identify the cause of an issue before attempting a fix.


## Why Error Messages Should Be Read Carefully

Error messages provide important information about what went wrong.

They can tell us:

* The type of error
* The file where the error occurred
* The line number involved
* Possible causes of the problem

Ignoring error messages often leads to wasted time and incorrect fixes.

### Example

Error:

```text
NameError: name 'username' is not defined
```

This tells us that the variable `username` was used before it was created.

---

## Reading Error Messages

When reading an error message:

1. Read the entire message.
2. Identify the error type.
3. Check the file name.
4. Check the line number.
5. Understand what the error is saying.

Example:

```text
TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

This means an integer and a string are being combined incorrectly.


## Reproducing a Bug

A bug should be reproduced before attempting a fix.

Reproducing a bug means performing the same steps that cause the issue so it can be observed consistently.

### Benefits

* Confirms the issue exists.
* Helps identify the root cause.
* Makes testing fixes easier.

### Example

Steps:

1. Open login page.
2. Enter invalid email.
3. Click Login.
4. Application crashes.

If these steps consistently produce the issue, the bug is reproducible.


## Using Logs

Logs are records generated by an application that show what happened during execution.

Logs help developers:

* Track system behavior
* Identify failures
* Understand application flow
* Investigate production issues

Example log:

```text
2026-06-20 10:15:22 ERROR User login failed: Invalid password
```

This helps identify exactly what happened and when.


## Using Print Statements

Print statements are simple debugging tools used to inspect values during program execution.

Example:

```python
name = "John"
print(name)
```

Output:

```text
John
```

Print statements help verify:

* Variable values
* Function execution
* Program flow


## Checking Assumptions

Many bugs occur because developers assume something is true without verifying it.

Always verify assumptions such as:

* Is the input valid?
* Is the API returning data?
* Is the database connected?
* Is the variable initialized?

### Example

Assumption:

```text
The user ID always exists.
```

Verification:

```python
print(user_id)
```

Checking assumptions prevents incorrect conclusions.


## Isolating the Problem

When debugging, isolate the section of code causing the issue.

Steps:

1. Divide the problem into smaller parts.
2. Test each part independently.
3. Identify where the failure occurs.

Benefits:

* Reduces complexity
* Speeds up troubleshooting
* Makes root cause identification easier


## Searching Documentation

Documentation often contains solutions, examples, and explanations.

Useful documentation sources include:

* Programming language documentation
* Framework documentation
* Library documentation
* API documentation

Before asking for help:

1. Read the error message.
2. Search official documentation.
3. Try recommended solutions.


## Explaining What Was Already Tried

When reporting an issue, explain the troubleshooting steps already completed.

Include:

* Error message
* Reproduction steps
* Logs collected
* Documentation reviewed
* Solutions attempted

Example:

```text
I received a TypeError on line 25.
I checked the variable types using print statements.
I reviewed the official Python documentation.
I attempted type conversion, but the error still occurs.
```

This helps others assist more effectively.


## Asking Clear Technical Questions

A good technical question should include:

### Problem

What is happening?

### Expected Result

What should happen?

### Actual Result

What actually happened?

### Error Message

Include the exact error.

### Steps to Reproduce

Explain how the issue occurs.

### What Was Tried

List troubleshooting steps already completed.

### Example

```text
I am building a login form in Python.

Expected:
The user should log in successfully.

Actual:
The application crashes.

Error:
NameError: username is not defined

Steps:
1. Enter username.
2. Click Login.

What I Tried:
- Checked variable declaration.
- Added print statements.
- Reviewed Python documentation.
```

This provides enough information for others to help efficiently.


# Debugging Checklist

Before asking for help, answer these questions:

1. What exactly is the error?
2. When does it happen?
3. Can I reproduce it?
4. What changed recently?
5. What logs are available?
6. What have I already tried?
7. What do I need help with?


# Key Takeaways

* Debugging is a systematic process, not guessing.
* Always read error messages carefully.
* Reproduce bugs before fixing them.
* Use logs and print statements to gather information.
* Verify assumptions with evidence.
* Isolate the problematic section of code.
* Search official documentation.
* Clearly explain troubleshooting steps already performed.
* Ask technical questions with enough context.
