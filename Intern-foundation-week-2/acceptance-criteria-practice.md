# Acceptance Criteria Practice

## Ticket 1: Add Item to Shopping List

**Issue Type:** Task

**Priority:** High

**Description:**
Develop functionality that enables users to add new items to their shopping list.

**Subtasks:**

* Create a structure to store shopping list items
* Accept item details from the user
* Validate the entered information
* Add the item to the shopping list
* Display the updated list

**Acceptance Criteria:**

* Users can add a valid item to the shopping list
* Empty item names are not accepted
* Newly added items appear in the list
* The application runs without errors

**Deliverable:**
A working feature that allows users to add items to a shopping list.

## Ticket 2: Remove Item from Shopping List

**Issue Type:** Task

**Priority:** High

**Description:**
Create functionality that allows users to remove items they no longer need from their shopping list.

**Subtasks:**

* Display available items in the shopping list
* Accept the item selected for removal
* Verify that the item exists
* Remove the item from the list
* Show the updated shopping list

**Acceptance Criteria:**

* Users can remove an existing item
* Removed items no longer appear in the list
* A message is displayed when an item cannot be found
* The updated list is shown after removal

**Deliverable:**
A working feature that removes items from a shopping list.

---

## Ticket 3: View All Items in Shopping List

**Issue Type:** Task

**Priority:** Medium

**Description:**
Implement a feature that displays all items currently stored in a user's shopping list.

**Subtasks:**

* Retrieve all items from the shopping list
* Organize items in a readable format
* Handle cases where the list is empty
* Test the display functionality

**Acceptance Criteria:**

* Users can view every item in the shopping list
* Items are displayed clearly and accurately
* A message is shown when the shopping list is empty
* The feature works without errors

**Deliverable:**
A working feature that displays all shopping list items.

---

## Ticket 4: Calculate Total Price of Selected Items

**Issue Type:** Task

**Priority:** High

**Description:**
Build functionality that calculates the total cost of selected shopping list items and presents the result to the user.

**Subtasks:**

* Store price information for items
* Allow users to select one or more items
* Calculate the combined price
* Display the total amount
* Test different calculation scenarios

**Acceptance Criteria:**

* Users can select multiple items
* The total price is calculated correctly
* The calculated amount is displayed clearly
* Invalid price values are handled appropriately
* The calculation completes without errors

**Deliverable:**
A working feature that calculates and displays the total price of selected items.

---

## Ticket 5: Display Friendly Error Messages for Invalid Input

**Issue Type:** Task

**Priority:** High

**Description:**
Create user-friendly error handling that informs users when invalid input is entered.

**Subtasks:**

* Identify possible invalid input scenarios
* Write clear and understandable error messages
* Display the correct message when an error occurs
* Prevent the application from crashing
* Test common error cases

**Acceptance Criteria:**

* Invalid input is detected successfully
* Users receive clear and helpful error messages
* Technical system errors are hidden from users
* The application continues running after an error occurs
* Error handling functions correctly in tested scenarios

**Deliverable:**
A working error-handling feature that displays friendly messages for invalid input.
