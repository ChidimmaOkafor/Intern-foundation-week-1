# Internet & Web Basics

## What happens when a user opens a website

When a user enters a website URL like www.example.com, several steps happen behind the scenes:

1. The browser acts as a client and sends a request to access the website.

2. The browser checks the DNS system to convert the domain name into an IP address.

3. Once the IP address is found, the browser sends an HTTP or HTTPS request to the server.

4. The server receives the request, processes it, and prepares a response.

5. The server sends back a response containing files such as HTML, CSS, and JavaScript.

6. The browser receives the response and renders the webpage so the user can see and interact with it.

## Client vs Server

- A client is a device or software that sends requests (e.g., browser or mobile app).
- A server is a system that receives requests and sends back responses.

## DNS

DNS (Domain Name System) translates domain names like google.com into IP addresses that computers understand.

## Hosting

Hosting is the process of storing a website or application on a server so it can be accessed over the internet.

## HTTP vs HTTPS

- HTTP is a protocol used to transfer data between client and server.
- HTTPS is the secure version of HTTP that encrypts data for safety.

## Request and Response

- Request: Sent from client to server asking for data.
- Response: Sent from server back to client with the requested data.

## APIs

An API allows different software systems to communicate and exchange data through defined rules, usually over HTTP/HTTPS.
